Normalisation Procedure

Exiqon miRNA Microarrays - Array 1 of 2

Overview

Normalisation is used to make the intinsities of the probes comparable to one another. Looking at the profile plot in the normalised data section, you can see how this works when changing from raw data to normalised data. A traditional two-colour array would involve initially background correcting the two colour intensities found on the arrays (Cy3 and Cy5), this is a form of noise filtration. Next normalising is carried out within arrays then between arrays. Finally we'd use the measurements between the Cy3 and Cy5 channels as our final expression set. Measurements are typically M (log2 Expression Ratio) or A (Average log2 expression)

Using the traditional approach to normalising a two-colour array was unsucessful. Generating a profile plot of the normalised expression set saw an unusual distinction between highly expressed probes and very low expressed probes. (As seen in the image above)

The result of the profile plot showed that Exiqon arrays do not follow the normal two-colour array paradgism. I used the ExiMiR package for normalisation which was specifically developed for these arrays. This uses spike- in probes, doesn't use backgroud correction (intentionally) and uses a medod of "Median-Polish" to normalise the arrays. Unusually, the final expression set is not derived from a measurement between dye intensities, but the green channel specifically is used.
